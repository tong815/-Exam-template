# Exam Template Project Workflow

## 项目定位

Exam Template 不是一个出卷器。

它的核心是：

```text
Exam JSON Schema
    ↓
Editor
    ↓
Preview
    ↓
PDF
```

其中：

- **JSON** 是唯一真实数据源（Single Source of Truth）
- **Editor** 是 JSON 编辑器
- **Preview** 是 JSON 可视化
- **PDF** 是最终输出

因此：

**内容存储在 JSON 中。**

编辑器和预览只是 JSON 的不同视图。

---

## 角色分工

### Teacher（用户）

负责：

- 决定考试目标
- 决定课程
- 决定难度
- 决定题型结构
- 提供原题
- 审核最终试卷

例如：

- G11 Functions Final
- 总分 70
- 90 分钟
- 学生基础较弱
- 覆盖：
  - Quadratic
  - Exponential
  - Trigonometric

---

### ChatGPT

负责：

- 根据考试要求生成 `exam-data.json`
- 将原题整理为 schema 结构
- 检查 schema 是否足够表达需求
- 判断是否需要扩展：
  - schema
  - renderer
  - editor
- 审查项目架构

原则：

**内容问题优先修改 JSON。**

---

### Cursor

负责：

- 修改代码
- 修改 schema
- 修改 renderer
- 修改 editor
- 修改打印样式
- 实现新功能

原则：

**不负责决定考试内容。**

---

## 日常工作流

### 情况 1：普通出卷

```text
Teacher
    ↓
描述考试需求
    ↓
ChatGPT
    ↓
生成 exam-data.json
    ↓
Open Project
    ↓
检查 Preview
    ↓
Save Project
    ↓
Print / Save as PDF
```

---

### 情况 2：使用已有原题

```text
Teacher
    ↓
提供原题
    ↓
ChatGPT
    ↓
整理为 exam-data.json
    ↓
Open Project
    ↓
检查 Preview
    ↓
Print PDF
```

---

### 情况 3：需要新表现形式

例如：

- 图像
- 表格
- 函数图
- 几何图
- 公式

流程：

```text
Teacher
    ↓
提出需求
    ↓
ChatGPT
    ↓
判断 schema 是否支持
```

**如果支持：**

仅修改 JSON

**如果不支持：**

生成 Cursor 修改指令

```text
Cursor
    ↓
扩展项目
```

之后：

```text
JSON
    ↓
Renderer
    ↓
Preview
```

## 判断规则

### 内容问题 → 修改 JSON

例如：

- 题目
- 分值
- 答案
- 标签
- Rubric

---

### 显示问题 → 修改 Renderer

例如：

- 表格显示
- 图片显示
- 函数图显示
- 打印格式

---

### 编辑不方便 → 修改 Editor

例如：

- 批量改分值
- 拖动排序
- 题目复制

---

### JSON 表达不了 → 修改 Schema

例如，新增：

- attachments
- rubricAllocation
- graph
- image
- table

---

## 文件管理

推荐目录：

```text
C:\Users\chena\Desktop\试卷json
```

例如：

```text
试卷json/
│
├─ G11 Functions/
│   ├─ Final 2026/
│   │   ├─ exam-data.json
│   │   ├─ exam.pdf
│   │   └─ notes.md
│   │
│   └─ Unit Test 1/
│
├─ G12 Advanced Functions/
│
└─ Physics/
```

在编辑器中：

1. **Open Project** — 打开 `exam-data.json`
2. 编辑后 **Save Project** — 直接覆盖原文件
3. **Print / Save as PDF** — 输出最终试卷

新建考试：先 **Load Profile** 搭骨架，再 **Save Project As** 保存到上述目录。

## GitHub 仓库职责

GitHub 仓库保存：

- Editor
- Renderer
- Schema
- Examples
- Documentation

**不建议保存：**

- 学校正式考试
- 实际学生卷
- 私有答案

真实考试建议保存在本地试卷目录。

---

## 核心原则

Exam Template 项目本质上是：

**Exam JSON Studio**

长期稳定的核心不是网页。

而是：

**`exam-data.json` schema**

未来：

- AI 出卷
- 题库
- PDF
- Web Editor
- Desktop Editor

都只是这个 schema 的**消费者（consumer）**。
