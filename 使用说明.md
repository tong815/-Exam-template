# Exam Template 使用说明

面向教师与日常使用者的操作手册。技术细节与 schema 文档见 [README.md](README.md) 与 [docs/schema-v1.md](docs/schema-v1.md)。

---

## 这是什么

**Exam JSON Studio** 不是自动出卷器，而是一套围绕 **`exam-data.json`** 工作的本地试卷工具：

```text
exam-data.json（试卷数据）
        ↓
   左侧 Editor（编辑）
        ↓
   右侧 Preview（预览）
        ↓
   Print / PDF（打印输出）
```

**记住一点：** 试卷内容保存在 JSON 文件里。左侧编辑区和右侧预览，只是同一份数据的两种看法。

## 快速开始

### 1. 打开工具

1. 用 **Chrome** 或 **Edge** 打开项目中的 `index.html`
2. 若需从 `templates/` 加载 starter template，建议用 VS Code **Live Server** 打开（双击 `file://` 也可用，内置回退数据仍可工作）

### 2. 打开或新建试卷

| 你想做的事 | 操作 |
|:-----------|:-----|
| 继续编辑已有试卷 | 工具栏 **打开项目** → 选择 `exam-data.json` |
| 从空白或模板开始 | **模板 (Template)** 选 Blank / G11 Functions Template → 编辑 → **项目另存为** |

### 3. 保存与打印

1. 改完后点 **保存项目**（直接覆盖原 JSON）
2. 右侧确认预览无误
3. 点 **打印 / 另存为 PDF**

## 界面说明

### 工具栏按钮

| 中文界面 | 英文界面 | 作用 |
|:---------|:---------|:-----|
| 打开项目 | Open Project | 从磁盘打开 `exam-data.json` |
| 保存项目 | Save Project | 覆盖保存当前项目；若尚未打开文件，会自动进入「另存为」 |
| 项目另存为 | Save Project As | 另存为新 JSON 文件 |
| 保存草稿 | Save Draft | 仅写入浏览器本地（刷新后可恢复），**不写磁盘** |
| 恢复默认 | Reset to Default | 恢复当前 starter template 的默认骨架 |
| 打印 / 另存为 PDF | Print / Save as PDF | 打印或导出 PDF（只输出右侧试卷） |

另有：**模板**（从模板新建）、**纸张**（Letter / A4）、**界面语言**、答案预览 / 教师备注切换。

### 当前项目状态

工具栏左侧显示 **当前项目：** 文件名。

| 显示 | 含义 |
|:-----|:-----|
| `exam-data.json` | 已打开项目，修改已保存 |
| `exam-data.json *` | 已打开项目，**有未保存修改** |
| `未保存草稿` | 尚未打开磁盘文件（可能来自 starter template 或浏览器草稿） |
| `未保存草稿 *` | 有未保存修改，且尚未关联磁盘文件 |

离开页面前若有 `*`，浏览器会提示「存在未保存修改」。请用 **保存项目** 写入磁盘；**保存草稿** 不能代替磁盘保存。

### 编辑区与预览区

| 区域 | 用途 |
|:-----|:-----|
| **左侧 Editor** | 改课程信息、标题、大题结构、每题题干/分值/答题行数等 |
| **右侧 Preview** | 学生卷实时预览；打印时只输出这一侧 |
| **顶部 Validation** | 数据结构校验（✅ / ⚠️ / ❌）；有错误仍可预览，正式出卷前建议修复 |

界面语言（中/英）**只翻译按钮和字段名**，不会翻译试卷正文。题目写什么语言，预览就是什么语言。

## 常用操作

### 打开已有试卷

1. 点击 **打开项目**
2. 在文件选择器中选 `exam-data.json`（例如 `试卷json\G11 Functions\Final 2026\exam-data.json`）
3. 在左侧修改，右侧即时预览
4. 完成后 **保存项目**

> 需 Chrome / Edge。Firefox / Safari 不支持直接读写项目文件，会提示换浏览器。

### 新建一份试卷

1. **模板** 选 **Blank Template** 或 **G11 Functions Template**
2. 在 **大题结构** 中调整题量、分值、答题行数
3. 填写 **考试元信息**（标题、时长、学校等）
4. 点击 **项目另存为**，保存到本地目录，例如：  
   `C:\Users\chena\Desktop\试卷json\G11 Functions\Final 2026\exam-data.json`
5. 之后每次用 **打开项目** / **保存项目** 继续编辑

### 编辑与保存

- 每次改动会自动写入浏览器 **localStorage**（防刷新丢失），这是草稿机制
- 正式保存到磁盘：点 **保存项目**
- 想备份另一份：点 **项目另存为**
- 误改结构想回到模板：**恢复默认**（会丢失当前未保存编辑，需确认）

### 打印 PDF

1. 选择 **Letter**（加拿大常用）或 **A4**
2. 确认右侧 Preview 排版
3. **打印 / 另存为 PDF** → 在系统对话框选「另存为 PDF」
4. 建议 PDF 与 JSON 放在同一文件夹，便于归档

### 从模板新建（Create From Template）

工具栏 **模板** 用于从 starter template 新建骨架（**不是**打开真实考试）。切换前会确认，避免覆盖未保存内容。

真实考试请用 **打开项目** 指向 `C:\Users\chena\Desktop\试卷json\...` 下的 `exam-data.json`。

## 文件放在哪里

### 推荐目录

本地试卷与 GitHub 上的**工具代码**分开存放。正式考试建议放在：

```text
C:\Users\chena\Desktop\试卷json\
```

### 目录示例

```text
试卷json/
│
├─ G11 Functions/
│   ├─ Final 2026/
│   │   ├─ exam-data.json    ← 试卷数据（核心）
│   │   ├─ exam.pdf          ← 打印出的 PDF（可选）
│   │   └─ notes.md          ← 命题笔记（可选）
│   │
│   └─ Unit Test 1/
│       └─ exam-data.json
│
├─ G12 Advanced Functions/
│
└─ Physics/
```

### 各文件是什么

| 文件 | 说明 |
|:-----|:-----|
| `exam-data.json` | **唯一数据源**：题目、分值、结构、答案占位等 |
| `exam.pdf` | 打印产物，方便发给学生或存档 |
| `notes.txt` / `notes.md` | 命题笔记，教师自行创建，工具不自动读写 |

### GitHub 上放什么

| 适合放在 GitHub | 不建议放在 GitHub |
|:----------------|:------------------|
| 编辑器、预览、打印代码 | 学校正式考试卷 |
| Schema 与示例 JSON | 含学生信息的卷子 |
| 文档 | 未公开的完整答案（若需保密） |

## 和 ChatGPT 一起用

你可以把 **描述需求 → 生成 JSON → 本工具预览/打印** 当作日常出卷方式。

### 你（教师）负责

- 考试目标、课程、难度、时长、总分
- 题型结构（选择 / 简答 / 应用等）
- 原题材料与最终审核

**示例需求：**

> G11 Functions 期末考试，70 分，90 分钟，学生基础偏弱，覆盖 Quadratic、Exponential、Trigonometric。

### ChatGPT 负责

- 按你的要求生成或整理 `exam-data.json`
- 把 Word / 纸笔原题整理成工具能识别的结构
- 判断现有 JSON 字段是否够用；不够时说明需要改工具哪一部分

### Cursor 负责（改工具本身）

- 改编辑器、预览、打印样式
- 扩展 JSON 结构（例如新的附件类型）
- **不**替你决定考什么、出什么题

---

## 三种常见出卷方式

### 方式 A：从零描述，AI 生成 JSON

1. 向 ChatGPT 描述考试要求
2. 得到 `exam-data.json`
3. 本工具 **打开项目** → 检查 Preview
4. **保存项目** → **打印 PDF**

### 方式 B：已有原题，AI 整理成 JSON

1. 把原题（Word、图片、手写）发给 ChatGPT
2. 整理为 `exam-data.json`
3. **打开项目** → 微调 → **保存项目** → 打印

### 方式 C：需要新的显示形式

例如：插图、表格、函数图、几何图、复杂公式。

1. 先说明需求，让 ChatGPT 判断 **现有 JSON 能否表达**
2. **能表达**：只改 JSON，在本工具里预览
3. **不能表达**：由 ChatGPT 写出给 Cursor 的修改说明，扩展工具后再继续用 JSON 编辑

## 出问题了改哪里？

按问题类型处理，不必记开发术语：

| 你想改的是… | 怎么做 |
|:------------|:-------|
| 题目文字、分值、答案、标签、Rubric | 在 Editor 里改，或让 ChatGPT 改 JSON |
| 预览/打印排版、图片怎么显示 | 属于工具显示层，需改项目代码（Cursor） |
| 编辑操作太麻烦（批量改分、复制题） | 属于编辑器功能，需改项目代码（Cursor） |
| JSON 里根本没有字段可描述你的需求 | 需扩展数据结构（Cursor + schema 文档） |

**简单原则：内容问题优先改 JSON；改 JSON 仍做不到，再改工具。**

## 核心观念（可选阅读）

本项目的长期核心是 **`exam-data.json` 的数据结构**，而不是某一个网页。

可以把它理解为 **Exam JSON Studio**：

- 现在的 Web 编辑器、预览、PDF，都是这份 JSON 的「消费者」
- 以后若有 AI 出卷、题库、桌面版，也应读写同一份 schema

因此：**稳定的是 JSON 格式；界面和打印方式可以换，数据不必丢。**

## 浏览器与限制

| 项目 | 说明 |
|:-----|:-----|
| 推荐浏览器 | Chrome、Edge（支持打开/保存项目文件） |
| 草稿 | 自动存 `localStorage`；**保存草稿** 手动触发同一机制 |
| 刷新页面 | 磁盘文件关联会丢失，需重新 **打开项目**；草稿内容通常仍可恢复 |
| 不做的事 | 不自动扫描文件夹、不上传云端、不替你做题库或 AI 出题 |
